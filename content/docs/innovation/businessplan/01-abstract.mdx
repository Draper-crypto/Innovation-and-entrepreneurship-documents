## 1. 项目背景

&emsp;&emsp;用 **时间 + 地点 + 对象 + 事件 + 问题** 的格式快速呈现 3–5 个案例，每个案例 100 字以内即可。为了加强视觉效果，建议配与案例数量一致的案例图（流程/场景/对比），参考案例如下：

- **[某年某月某日 某地] [对象/人群]** 在 **[场景]** 中出现 **[事件/行为]**，导致 **[指标 A/B]** 异常（如：停机率增加 23% / 投诉率 2.1%）。
- **[某年某月某日 某地] [对象/系统]** 在 **[流程节点]** 发生 **[故障/低效]**，带来 **[损失/风险]**（如：人均处理时长 48→71 分钟）。
- **[某年某月某日 某地] [对象/岗位]** 受 **[客观约束/制度缺口]** 影响，**[关键目标]** 难达成（如：合规缺失导致返工率 5.8%）。

&emsp;&emsp;在案例编写完成后应当对案例中体现的问题（痛点）进行小结。

<Callout type="info" title="提示：">
  - 为了防止在比赛过程中评委询问案例的来源、真实性，你可以在完成本节编写后，使用小字进行注释，标注数据的来源网站名、网址，例如
  `[1] 国家统计局：网址`； 
  - 本节小结时，可以使用加粗红色字去突出展示重点内容，样式可以自定义，无需强制按照本文去标注。
  - 痛点数量不要多，一般三至四个即可。
</Callout>

## 2. 项目概况

本部分是项目计划书最重要的部分，本部分需要以最少的篇幅去讲清楚你的项目，具体包括：

- 项目是做什么的？
- 项目解决什么问题？
- 项目创造什么价值？
- 目标客户有哪些？

首先需要一句话概述项目，你可以按照下述模板进行修改：

```text
格式：

因为 [目标用户/场景] 长期面临 [首要痛点]，我们提供 [核心方案/产品/服务]，用于 [关键环节]，使 [可量化收益A/B]，相较 [主要替代] 的优势是 [差异化1/2]。

具体案例：

因为 工程起重设备（塔吊/桥吊/门吊）高风险工况下 长期面临 误报/漏报与事后处置滞后，我们提供 “CraneGuard 多传感融合 + 边云协同监测预警系统”，用于 吊装作业的实时风险识别、预测性维护与取证闭环，使 重大隐患识别准确率 ≥95%，误报率 ≤3%，停机时长 下降20%，相较 “单传感器 + 人工巡检” 的优势是：
（1）对风载/冲击/偏载等边界工况的鲁棒识别；
（2）事前预警 + 取证追溯的管理闭环。
```

接着参考以下表格，对应填写：

|   类别   | 内容                                                                                                 |
| :------: | :--------------------------------------------------------------------------------------------------- |
| 项目名称 | 填写项目的名称                                                                                       |
| 项目概述 | 上方编写的一句话概述项目                                                                             |
| 业务概述 | 填写你成立的公司**主要做什么**，简述产品的功能作用，具体解决什么问题                                 |
| 商业模式 | 谁付费、怎么付费：一次性/订阅/服务/硬件+SaaS 等                                                      |
| 服务模式 | 讲清楚通过什么服务帮助客户解决问题，即如何交付与落地，注意与产品概况不同，此处侧重服务               |
| 盈利简况 | 填写项目怎么盈利，通过什么方式盈利                                                                   |
| 知识产权 | 此处简述已有/在申报著作权、软件著作权、商标、外观专利、实用新型专利、发明专利，格式统一为：编号+名称 |

## 3. 应用前景

简明扼要的讲清楚以下几点：

- 现在市场是什么样的，预估客户有多少；
- 从研报、调研、咨询等渠道，判断未来客户有多少；
- 有没有国家政策扶持，有哪些政策扶持，未来还会不会有政策利好。


举例说明，仅供参考：

- 根据某协会的数据，2024年设备总量为47.8万台，同比增长12%，其中塔吊为24.4万台，同比增长13%。
- 目前国内的起重设备智能化处于较为落后的阶段，主要是因为设备的生产成本较高，而智能化设备的生产成本较高。
- 某前瞻研报指出，未来10年，中国的起重设备智能化将达到100%。
- 某规划中指出，需要大力发展起重设备智能化转型。
- 未来，随着智能化设备的成本下降，智能化设备的普及率将提高，会带动产品的销量增加。
