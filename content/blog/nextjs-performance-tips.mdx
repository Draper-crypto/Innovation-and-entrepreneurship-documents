在大型前端项目中，性能体验往往决定了留存率与转化率。本文以清单形式列出 Next.js 常见的性能优化策略与落地示例。

## 1. 渲染策略选择（SSR/SSG/ISR）

- 落地建议：以 SSG 为优先，动态数据采用 ISR；仅对强依赖实时性的页面使用 SSR。
- 验证方式：使用 Lighthouse 与 Chrome Performance 分析首屏时间。

## 2. 资源分割与按需加载

- 通过 `next/dynamic` 实现组件级代码分割，延迟加载不影响首屏的模块。
- 为第三方依赖配置 `modular import` 或替代更轻量的库。

## 3. 图片优化

- 使用 `next/image`，结合 `sizes`、`priority`、`placeholder` 提升 LCP。
- 为列表图片开启懒加载，控制解码时机。

## 4. 缓存与边缘计算

- 公共数据放入 CDN，接口响应设置恰当的 `Cache-Control`。
- 结合 Vercel Edge / 中间层缓存，减少源站压力。

## 5. 监控与回归

- 建立性能基线（如 LCP < 2.5s），在 CI 中集成性能回归检查。

> 小结：先定位瓶颈，再做“必要的优化”。工具优先，经验其次。
